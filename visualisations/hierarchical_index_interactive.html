üí° What is Hierarchical Indexing?

Hierarchical indexing (MultiIndex) allows you to have multiple index levels on an axis. It's like organizing a library with categories, subcategories, and books - you can navigate at any level! This enables you to work with higher dimensional data (3D, 4D, etc.) in a familiar 2D DataFrame structure.

‚ùå Regular Index (Flat)

Index Value
Item_1 42
Item_2 87
Item_3 23

One level - Can only access by exact key

‚úÖ Hierarchical Index (MultiIndex)

Level 0 Level 1 Value
Category A 1 42
Category A 2 87
Category B 1 23

Multiple levels - Can access by outer level, inner level, or both!

‚öôÔ∏è Common Operations

Hierarchical indexes support many useful operations:

üîÑ Swap Levels

Exchange the order of index levels

frame.swaplevel('key1', 'key2')

üìä Sort Index

Sort by index levels

frame.sort_index(level=1)

üìà Group By Level

Aggregate data by specific level

frame.groupby(level='key1').sum()

üé® Cross Section

Select single level value (xs)

frame.xs('a', level='key1')

üí° Key Takeaways

  • ‚úÖ Partial Indexing: Select all data for an outer level key
  • ‚úÖ Multi-level Selection: Navigate through any level of hierarchy
  • ‚úÖ Reshape Operations: Stack, unstack, pivot with ease
  • ‚úÖ Efficient Storage: Represent higher dimensional data in 2D
  • ‚úÖ Flexible Grouping: Aggregate by any level or combination of levels

üí° Pro Tip: Hierarchical indexing is especially useful for time series data, panel data, and any multi-dimensional dataset that needs to be stored efficiently!
